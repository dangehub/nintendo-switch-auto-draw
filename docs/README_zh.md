# Nintendo Switch Auto Draw (NS 自动画图)

> [!NOTE]
> **Vibe Coding / AI 辅助声明**
> 请悉知：本项目的核心架构、功能逻辑以及大部分代码生成均是通过与 AI 结对编程（Vibe Coding）完成的。

这是一个基于 Web Serial API 和 2wiCC (Switch Control via USB v2) 的任天堂 Switch 自动绘图工具。可以直接在浏览器中运行，无需安装任何客户端软件或配置复杂的 Node.js 环境。
Auto Draw 将图片转化为 2WICC 能识别的命令，然后模拟 NS 手柄发送绘图命令。

## 🌟 特性

- **全自动绘画**：导入图片后，自动进行量化、抖动处理、颜色提取，并在游戏中自动换色、全自动进行蛇形智能填色。
- **智能寻路扫描**：利用包围盒算法跳过透明和空白像素，极大缩短绘图耗时。
- **纯网页无端运行**：借助 Web Serial API 直接通过 USB 控制单片机，即点即用。
- **直观的编辑体验**：随时进行移动、缩放、擦除。

### 软件使用教程

- 点击访问：[NS Auto Draw](https://draw.qudange.top/)
- 将 2WICC 一端插入 PC，一端插入 NS，在 Auto Draw 点击连接 ![软件连接](软件连接.png)
- 根据需求在右上角切换不同游戏的配置![切换配置](切换配置.png)
- 在网页左侧添加想要转绘的图片
- 通过预设或自定义调整参数
   - 画布宽度：游戏内画布的宽度
   - 画布高度：游戏内画布的高度
   - 颜色数量：将上传的图片转换为多少颜色的绘图（splatoon3为单色，朋友收集等多色绘图游戏则可以由原图色彩复杂度自行调整）
   - 起始X：进入画板后，画笔默认的位置（一般是画面中心，即宽度/2）
   - 起始Y：进入画板后，画笔默认的位置（一般是画面中心，即高度/2）
   - 图像缩放：原图的缩放倍率
   - 图像偏移X：预览图在画布上的X轴偏移量
   - 图像偏移Y：预览图在画布上的Y轴偏移量
   - 黑白阈值（单色）：用于控制将原图转换为单色预览图的强度
   - 按键帧数（pf）：控制给 Switch 发送“按键按下”状态的持续时间（单位：帧，1帧约等于16.6毫秒）。如果数值太小，Switch 可能无法识别按键导致漏画；如果数值太大，整体绘图时间会显著变长。
   - 释放帧数（rf）：控制两次按键之间的“按键松开”等待时间。同样是为了防止输入过快导致 Switch 丢帧。如果您在绘制时发现有断线或漏点，可以尝试稍微调高 pf 和 rf。
   - 抖动算法：当图片的颜色被压缩（例如彩色转为 Splatoon 3 的黑白两色）时，用来处理过渡色的算法。
      - Flody-Steinberg：误差扩散算法。会用密集的黑白噪点来模拟出“灰色”等过渡颜色，保留更多原图的亮度和细节。**非常适合照片、渐变色和插画**。
      - 无：直接将像素归为最近的颜色（纯黑或纯白），不产生任何噪点过渡。图像边缘非常锐利干净。**非常适合简笔画、线稿、Logo 和文字**。

## 如何部署项目

### 使用我部署好的页面

我已经使用Vercel托管了本项目，如果您不想折腾，可以直接点击访问：[NS Auto Draw](https://draw.qudange.top/)

### 自行部署

本项目完全是一个静态前端网页。为了让小白用户也能零门槛使用，您可以直接使用 **GitHub Pages** 功能将它部署在网上：

1. 将本项目 Fork 或克隆到您的 GitHub。
2. 在您的 GitHub 仓库中，进入 `Settings` -> `Pages`。
3. 在 `Source` 下方，选择 `Deploy from a branch`。
4. Branch 选择 `main` (或 `master`)，文件夹选择 `/(root)`，点击 Save。
5. 等待一两分钟后，GitHub 会生成一个您的专属链接（例如 `https://您的用户名.github.io/nintendo-switch-auto-draw/`）。
6. **打开这个专属链接，即可连接单片机进行画图！完全无需下载和配置环境。**

> **注意**：由于浏览器的安全限制，Web Serial API 必须在 HTTPS 环境下才能调用。所以通过 GitHub Pages 访问是最佳的选择。

## 🛠 本地开发与测试

如果您希望在本地进行二次开发：

1. 克隆本项目到本地。
2. 由于直接双击打开 `index.html` (使用 `file:///` 协议) 会触发跨域限制，且不支持 Web Serial API。
3. 请在项目根目录下启动一个本地 Web 服务，例如：
   ```bash
   npx serve -l 3456
   ```
4. 通过 `http://localhost:3456` 进行访问和调试。

## 📁 目录结构

- `index.html` - 主绘图控制面板
- `css/` - 样式表
- `js/` - 核心逻辑模块（图片量化处理、绘制引擎算法、串口通讯管理器）
- `tests/` - 开发者用的硬件调测与发包测试页面
